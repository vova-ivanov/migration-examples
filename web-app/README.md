# Lambdas and other resources not surfacing their code

Import discovers the full infrastructure graph — the REST API, its resources and methods, the DynamoDB table, the S3 bucket, the IAM role, and the Lambda function — but the function's handler contains real application logic: routing between GET, POST, and DELETE, reading and writing DynamoDB, and formatting responses. None of that business logic is part of the discovered state. A migration that relies only on what discovery returns will produce a correctly wired Pulumi program with a placeholder or stub handler; the actual application code must be retrieved from the source repository and reintegrated. The more significant the Lambda logic, the larger the gap between a successful import and production-ready migrated code.
