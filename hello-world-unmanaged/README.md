# Unmanaged resources are invisible to stack-based discovery

This application is identical in function to `hello-world`, but every resource — the IAM role, Lambda function, and API Gateway — was created directly through the AWS CLI with no CloudFormation stack involved. Stack discovery tools enumerate stacks and their managed resources; they have no mechanism to find infrastructure that was never declared in a stack. The IAM role, Lambda, and API Gateway here are completely invisible to any discovery pass, even though they are live and serving traffic. Migrating setups like this requires either manually identifying the resources or using a broader account-level resource scan, and in either case the relationships between resources must be inferred rather than read from a template.
