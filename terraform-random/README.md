# Terraform resources being treated the same as unmanaged ones, and being invisible

Unlike CloudFormation or ARM, Terraform does not create a dedicated "stack" object in AWS — its resources are indistinguishable from hand-created ones when viewed through the AWS API. A stack-discovery tool will surface the S3 bucket and Lambda function here, but it has no way to know they are Terraform-managed and will treat them the same as unmanaged resources, providing no grouping or dependency information. Compounding this, several resources in this configuration (`random_id`, `random_pet`, `time_static`, `tls_private_key`) are local Terraform constructs that make no AWS API calls and leave no footprint whatsoever — a scan will never see them, yet the deployed infrastructure depends on the values they produce.
